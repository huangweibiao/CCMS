# 企业级费控管理系统详细设计文档

## 一、系统目标与范围

### 1.1 核心目标

构建覆盖企业费用全生命周期的管理系统，实现费用申请（报销/预支/借款）、审批流转、预算控制、费用合规校验、财务入账、数据分析一体化管理。

### 1.2 系统边界

|包含范围|不包含范围（可扩展）|
|---|---|
|员工报销、审批流、预算控制、发票管理、财务对接（导出/接口）|完整ERP（如总账）、税务系统|
## 二、系统架构设计

### 2.1 总体架构（分层单体）

```Plain Text

┌─────────────────────────────────────────────────────────┐
│                      前端层                              │
│         Web端(React/Vue) + 移动端(H5/小程序)            │
├─────────────────────────────────────────────────────────┤
│                      表现层（Controller/API）            │
├─────────────────────────────────────────────────────────┤
│                      应用层（Application Service）       │
├─────────────────────────────────────────────────────────┤
│                      领域层（Domain）                    │
├─────────────────────────────────────────────────────────┤
│                      基础设施层（Infrastructure）        │
├─────────────────────────────────────────────────────────┤
│                      服务层                              │
│   权限服务 │ 通知服务 │ 附件服务 │ 规则引擎 │ 工作流     │
├─────────────────────────────────────────────────────────┤
│                      数据层                              │
│       MySQL(主库) + Redis(缓存) + 本地磁盘(附件)         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心模块划分

|模块|核心功能|
|---|---|
|用户与组织模块|用户、部门、角色、岗位管理，权限分配|
|预算管理模块|预算编制、调整、控制、分析、执行跟踪|
|费用申请模块|事前申请、借款申请、申请单跟踪、状态管理|
|费用报销模块|报销单填报、发票管理、费用分摊、借款核销|
|审批模块|审批流配置、待办/已办处理、审批记录、流程引擎|
|发票模块|发票识别（OCR）、验真、信息管理|
|财务模块|凭证生成、付款管理、财务对接|
|报表模块|费用分析、预算执行、部门统计、BI分析|
|配置模块|字典管理、规则配置、流程配置|
|消息通知模块|待办提醒、审批通知、状态变更通知|
## 三、核心业务模型设计

### 3.1 核心实体关系

```Plain Text

User → Dept → Expense(Apply/Reimburse) → ExpenseItem → Invoice
           ↓                          ↓
     Budget(Main/Detail)        ApprovalInstance → ApprovalNode/Record
           ↓
     BudgetAdjust
```

### 3.2 状态机设计（费用单核心流转）

```Plain Text

草稿 → 审批中 → 已通过 → 待支付 → 已支付
             ↘ 已驳回 ↗
                ↓
               作废
```

## 五、核心流程设计

### 5.1 费用报销核心流程

```Plain Text

填单（草稿）→ 预算校验（冻结预算）→ 提交审批（审批中）→ 部门审批 → 财务审批 → 出纳支付 → 完成（已支付）
                ↓                      ↓                ↓          ↓          ↓
             预算不足               审批驳回          审批中     审批中     待支付
              终止                   可修改重提
```

### 5.2 预算控制逻辑

```Plain Text

申请/报销金额 ≤ 预算余额（预算总额 - 已用金额 - 冻结金额）
```

### 5.3 审批流匹配逻辑

```Plain Text

根据业务类型（申请/报销）+ 金额区间 + 部门 + 费用类型 匹配最优审批流配置
```

### 5.4 借款核销流程

```Plain Text

报销单提交 → 检测借款人未还清借款 → 自动计算可抵扣金额 → 生成还款记录 → 扣减借款未还金额 → 剩余金额支付
```

## 六、关键业务规则

|规则项|规则内容|
|---|---|
|预算控制|超标单据无法提交，需先申请预算调整；预算扣减采用乐观锁保证并发安全|
|发票验真|发票金额≥1000元需自动验真；验真不通过的发票无法提交报销|
|审批路径|根据金额、费用类型、部门自动匹配审批流；支持固定流程、条件分支、动态审批人（直属上级）|
|借款核销|借款需在报销时自动抵扣；借款超期未还系统自动提醒|
|费用合规|餐补不超过100元/天；交通费用需匹配行程时间；无发票费用需特殊审批|
|数据隔离|仅能查看本部门及授权的费用数据；管理员可查看全量数据|
